# [Настройки сайта](settings-for-resource)

Настройки сайта позволяют управлять доменом и параметрами подключения: DNS-записями, SSL-сертификатом и перенаправлениями. Здесь же можно удалить сайт при необходимости.

Чтобы открыть настройки сайта, перейдите на страницу  [«Мои сайты»](https://my.rt-solar.ru/resources). В строке нужного сайта откройте меню действий `«⋯»` и выберите пункт **«Настройки»**.

![Open site settings from My sites page (95%)](https://img.solarspace.pro/docs/saas/resources/my-sites-open-settings.png "Открытие настроек сайта со страницы «Мои сайты»")

На вкладке «Домен» собраны основные настройки, связанные с доменом сайта: DNS-записи, сертификат и поведение перенаправлений. Здесь же можно удалить сайт при необходимости.

![Domain tab and settings panel(95%)](https://img.solarspace.pro/docs/saas/resources/site-domain-tab-settings-panel.png "Вкладка «Домен» и панель настроек сайта")

На вкладке «Домен» доступны:
- [Настройки DNS]([249#dns-settings]) — настройка DNS A-записи для перенаправления трафика на защиту и DNS TXT-записи для подтверждения владения доменом.
- [SSL-сертификат]([249#ssl-certificate]) — просмотр информации о бесплатном сертификате Let’s Encrypt для домена.
- [Редирект с WWW]([249#redirect-www]) — автоматическое перенаправление на основной адрес сайта независимо от наличия `www`.
- [Редирект на HTTPS]([249#redirect-https]) — автоматическое перенаправление с HTTP на защищённый HTTPS.
- [Удаление сайта]([249#delete-site]) — удаление сайта и отключение подключенных сервисов.

### [Настройки DNS](dns-settings)

В блоке «Настройки DNS» настраивается:
- перенаправление трафика на защиту (через DNS A-запись);
- подтверждение владения доменом (через DNS TXT-запись).

Если вы уже выполняли эти шаги в рамках Быстрого старта, повторно настраивать их не нужно:
- перенаправление трафика: [Постановка домена под защиту]([300#configure-a-record]);
- подтверждение владения доменом: [Верификация домена]([300#domain-verification]).

> **INFO**
> #### Важно
> В большинстве случаев достаточно настроить DNS A-запись и перевести трафик на защиту — этого хватает для подключения защиты и подтверждения домена.  
> DNS TXT-запись используется, если нужно подтвердить владение доменом без перенаправления трафика (например, чтобы использовать только сервисы сканирования).

Нажмите «Настроить» и следуйте подсказкам в окне «Верификация домена».

![DNS settings card(50%)](https://img.solarspace.pro/docs/saas/resources/site-dns-settings-card.png "Блок «Настройки DNS»")

Откроется окно «Верификация домена» с двумя вкладками:
- «Перенаправление трафика для подключения защиты» — на этой вкладке отображается значение DNS A-записи, которое нужно добавить у регистратора/хостинг-провайдера.

![Domain verification modal traffic forwarding tab(50%)](https://img.solarspace.pro/docs/saas/resources/domain-verification-modal-traffic-forwarding.png "Окно «Верификация домена»: перенаправление трафика")

- «Верификация домена для подтверждения владения доменом» — на этой вкладке отображается значение DNS TXT-записи для подтверждения владения доменом.

![Domain verification modal ownership verification tab(50%)](https://img.solarspace.pro/docs/saas/resources/domain-verification-modal-ownership-verification.png "Окно «Верификация домена»: подтверждение владения доменом")

> **INFO**
> #### Обратите внимание
> Если после настройки DNS ваш сайт находится в статусе «Не подтвержден», проверьте возможные причины в статье [Ошибки при верификации и перенаправлении трафика]([268]).

### [SSL-сертификат](ssl-certificate)

Для домена сайта автоматически выпускается бесплатный SSL-сертификат Let’s Encrypt. Сертификат используется для работы сайта по HTTPS и шифрования соединения.

Подробная инструкция по подключению и проверке SSL-сертификата приведена в статье [Домен и SSL]([213]).

Чтобы посмотреть информацию о сертификате в личном кабинете, откройте вкладку «Домен» и в строке домена нажмите на иконку настроек рядом с полем SSL. В открывшемся окне отображаются основные сведения о сертификате и срок его действия.

![SSL certificate details(95%)](https://img.solarspace.pro/docs/saas/resources/ssl-certificate-details[1].png "Информация о SSL-сертификате")

### [Редирект с WWW](redirect-www)

Перенаправляет обращения к сайту, введённые с префиксом `www` (например, `www.example.com`), на основной домен без `www` (`example.com`).  
Функция помогает использовать единый адрес сайта и снижает риск появления дублей страниц в поисковой выдаче. При отключённой настройке обращения с `www` могут завершаться ошибкой «Страница не найдена».

Рекомендуется включать редирект после подключения защиты (после настройки DNS A-записи) в разделе [Постановка домена под защиту]([300#configure-a-record]).

---

### [Редирект на HTTPS](redirect-https)

Перенаправляет все запросы с небезопасного протокола HTTP на HTTPS.

Редирект работает только при наличии SSL-сертификата. Если SSL-сертификат для домена не добавлен, перенаправление выполняться не будет.

После включения редиректа:
- запросы к `http://` будут автоматически перенаправляться на `https://`;
- сайт будет открываться только по защищённому протоколу.

Рекомендуется включать редирект на HTTPS после [добавления SSL-сертификата]([213#add-ssl-certificate]).

### [Удаление сайта](delete-site)

Для удаления сайта выполните следующие действия:
1. Нажмите кнопку «Удалить сайт».
2. В появившемся окне нажмите кнопку «Да, удалить сайт».
3. После подтверждения действия появится сообщение об успешном удалении. Сайт будет исключен из списка всех сайтов на странице [«Мои сайты»]([246]), а все его сервисы будут отключены.

> **WARNING**
> #### Важно
> Удаление сайта приведет к отключению всех его сервисов.

# [Настройки сайта](settings-for-resource)

Чтобы открыть настройки ресурса, перейдите на страницу ["Мои ресурсы"]([246]), нажав на иконку папки в левом меню с любой страницы личного кабинета.

Нажмите на три точки в правой части строки нужного ресурса. Откроется выпадающий список.

![Settings on resources page(95%)](https://img.solarspace.pro/docs/partners/resources/settings-on-resources-page.png "Настройки на странице ресурсов")

Выберите пункт "Настройки", либо просто нажмите на нужный ресурс.

На странице настроек ресурса вы можете:
- [Добавить домен]([249#adding-new-domain])
- [Удалить ресурс]([249#delete-resource])

### [Добавление доменов](adding-new-domain)
Если у вашего ресурса есть еще домены, которые вы хотите добавить для ресурса, выполните следующие действия:

![Added new domain(95%)](https://img.solarspace.pro/docs/partners/resources/added-new-domain.png.png "Добавление нового домена")

<ol>
  <li>Нажмите на кнопку «Добавить».</li>
  <li>В появившейся строке введите имя домена, соответствующее требованиям:
    <ul>
      <li>Поле домена не должно быть пустым.</li>
      <li>Домен зарегистрирован в системе доменных имен (DNS).</li>
      <li>Домен уникальный.</li>
      <li>Домен имеет корректный формат (например, example.com).</li>
      <li>Адреса доменов должны совпадать.</li>
    </ul>
  </li>
  <li>При корректном вводе доменного имени кнопка «Сохранить» станет активной. Нажмите на неё для добавления нового домена.</li>
</ol>

### [Верификация](verification)
Для верификации ресурса выполните следующие действия: 

1. В разделе настроек ресурса нажмите на кнопку "Верификация".
2. В появившемся окне скопируйте представленную DNS-TXT запись Solar Space.
3. Далее следуйте инструкциям, аналогичным процессу верификации после создания ресурса.

![Click on verification(95%)](https://img.solarspace.pro/docs/partners/resources/click-on-verification.png "Верифицировать ресурс со страницы ресурса")

> **SUCCESS**
>При успешной верификации будет соответствующая пометка "Ресурс и все его домены верифицированы".


### [Удаление ресурса](delete-resource)

Для удаления ресурса выполните следующие действия:
1. Нажмите кнопку "Удалить ресурс".
2. В появившемся окне нажмите на кнопку "Удалить" для удаления ресурса.
3. После подтверждения действия появится сообщение об успешном удалении. Ресурс будет исключен из списка всех ресурсов на странице ["Мои ресурсы"]([246]) и все его сервисы будут отключены.

> **WARNING**
> #### Важно
> Удаление ресурса приведет к отключению всех его сервисов
