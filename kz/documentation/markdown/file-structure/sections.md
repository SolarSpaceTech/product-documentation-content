# Бөлімдерді жасау

Бөлімдерді жасау документацияны ұйымдастыруда маңызды қадам болып табылады. Бөлімдер ақпаратты таратуға көмектеседі, оны пайдаланушыларға толық болуы және түсінуіне мүмкіндік береді. Осы нұсқауда біз бөлімдерді жасау процесін, олардың metadata арқылы орнатуын және пайдаланушы интерфейсін қосымшауын толық талдау етеміз.

<br/>

## [Бөлімдердің құрылымы](structure)

### Директорияларды бөлімдер ретінде қолдану

- **Әр бір директория** файл жүйесінде сайттың бір арнайы бөлімін көрсетеді.
- **Қосылған директориялар** бөлімдер мен бөлімдердің іерархиялық құрылымын жасауға мүмкіндік береді.

### `metadata.md` файлдары

- **Бөлім сипаттамасы**: Әр бір бөлімде `metadata.md` файлы болуы керек, бөлімдің метадеректерін қамтықтайтын.
- **Метадеректер пішімі**: Метадеректер үш түсті тире таңбалар арасында көрсетіледі.

<br/>

## [Бөлімдердің метадеректері](metadata)

Бөлімдердің метадеректері олардың көрсетуі мен сайттағы жұмысын анықтайды.

### `metadata.md` файлдарының құрылымы

Бөлім үшін `metadata.md` файлының үлгісінің мисалы:

```md
---
displayName: Бөлім атауы
order: Тәртіп нөмірі
published: true немесе false
---
```

### Метапараметрлер түсінігі

- **displayName**: Бөлім атауы, мәзірде және нан тіркемелерінде көрсетіледі.
- **order**: Бөлімді мәзірде көрсету тәртібі, оның орнындағы басқа бөлімдерге қарағанда.
- **published**: Бөлімді сайтта көрсету керек болса, true дегенді немесе жоқ. false немесе жоқ болса, бөлім көрсетілмейді.

<br/>

## [Бөлімді жасау процесі](process)

### 1 қадам: Директория жасау

- **Жаңа директория жасаңыз** файл жинағындағы қажетті орынға, жаңа бөлімді көрсететін жаңа директорияны жасаңыз.
  - Мысал: `documentation/new-section`

### 2 қадам: `metadata.md` файлын қосу

- **`metadata.md` файлын жасаңыз** жаңа директория ішінде.
- **Метадеректерді толтырыңыз** сіздің қажеттіліктеріңізге сәйкес.

```md
---
displayName: Жаңа бөлім
order: 50
published: true
---
```

### Қадам 3: Мақалаларды қосыңыз (міндетті емес)

- **Мақалалар файлдарын қосыңыз** `.md` кеширімімен бөлім директориясына.
- **Мақалалардың метадеректерін баптау** талаптарға сәйкес.

### Қадам 4: Қосымша бөлімдерді жасау (міндетті емес)

- **Бөлім ішінде қосымша қалталарды жасаңыз** қосымша бөлімдерді жасау үшін.
- **Әр бір қосымша қалтаға `metadata.md` қосыңыз** сәйкес метадеректерімен.

<br/>

## [Бөлімдердің пайдаланушы интерфейсіне тәсірі](impact)

### Сайт мәзінде

- **Бөлімдерді көрсету**: Мәзінің құрылымы мен бөлімдердің метадеректеріне негізделген.
- **Тәсір салатын өрістер**:
  - `displayName`: Мәзіде бөлім атауын белгілейді.
  - `order`: Бөлімдерді көрсету тӱрін белгілейді.
  - `published`: `false` немесе жоқ болса, бөлім көрсетілмейді.

### Нанымдар

- **Жолды құру**: Нанымдар коренді бөлімнен ағымдағы мақалаға дейін жолды көрсетеді.
- **`displayName` пайдалануы**: Бөлімдер мен мақалалар атаулары `displayName` өрісінен алынады.

### Басты бет және басқа элементтер

- **Шектеулі тәсір**: Бөлімдер мазмұнын ұйымдастыруға тәсір етеді, бұл интерфейс элементтерінде көрсетіледі.

<br/>

## [Бөлімді жасау туралы өріс](example)

### Қадам 1: "Мисалдар" бөлімін жасау

- **Директория**: `documentation/examples`
- **`metadata.md` файлы**:

```md
---
displayName: Мисалдар
order: 20
published: true
---
  ```

### Қадам 2: Бөлімде мақала қосу

- **Мақала файлы**: `documentation/examples/sample.md`
- **Мақала метадеректері**:

```md
---
id: 100
title: Мақала мисалы
displayName: Мақала мисалы
order: 10
published: true
---
```

### Қадам 3: Көрсету тексеру

- **Сайт мәзісі**: "Мисалдар" бөлімі менюде 20-реттік орында көрсетіледі.
- **Нанымдар**: "Мақала мисалы" мақаласын көру кезінде нанымдар жолы:
"Документация > Мисалдар > Мақала мисалы".

<br/>

## [Маңызды ескертулер](notes)

- **`order` уникалдығы**: Көрсетулердің қайталануларын алу үшін реттік нөмірлерді сақтаңыз.
- **`published` өрісі**: Бөлімдер мен мақалалардың көрсетілуін бұл өріс арқылы бақылаңыз.
- **Консистенттік**: Файл жүйесінің құрылымының мазмұнылық ұйымдастыруын тексеріңіз.
