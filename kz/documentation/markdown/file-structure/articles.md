# Мақалаларды жасау

Осы мақалада біз документациялар үшін мақалаларды жасау процесін қысқаша талқылау қыламыз, олардан HTML-беттер файл жинағындағы `*.md` файлдарынан жасалады.
Сіз мазмұнды ыңғайлы ұйымдастыра аласыз, пайдаланушы интерфейс элементтерін көрсету мен пайдаланушыларға ыңғайлы навигацияны ұсынуға мүмкіндік бересіз.

<br/>

## Файл жинағы

Әрбір мақала соответствующа бөлімнің каталогында орналасқан `.md` кеши бар файлмен көрсетіледі. Файл жинағы бөлімдер мен бөлімдердің һиерархиясын көрсетеді.

- **Негізгі каталог**: тілдердің каталогтарын қамтықтайды (`ru`, `en` және т.б.).
- **Тіл каталогы**: тіл мета-деректері мен басты беттің параметрлерін көрсететін `metadata.md` файлын қамтықтайды.
- **Бөлімдер және бөлімдер**: тіл каталогы ішінде каталог түрінде орналасқан, олардың өзінде metadata.md бар болуы мүмкін.
- **Мақалалар**: бөлімдердің каталогтары ішінде орналасқан `.md` файлдар.

<br/>

## Мақала мета-деректері

Әр файл мақаласының жоғары жағында мақала мета-деректерін **YAML** пішінінде үш тире (`---`) арасында көрсету қажет. Бұл мета-деректерлер интерфейс элементтерінде мақаланы көрсетеді.

### Мақала мета-деректерінің мисалы

```markdown
---
id: 42
title: Пайдаланушының нұсқаулары
displayName: Нұсқаулар
order: 10
published: true
historyName: Тарихтағы нұсқаулар
historyDescription: Пайдаланушының қысқаша сипаттамасы
category: Бастау
categoryName: Жылдам бастау
categoryDescription: Жылдам жұмыс істеу туралы мәлімет алыңыз
categoryOrder: 1
categoryIcon: /icons/start.svg
headerName: Шапкада нұсқаулар
headerOrder: 20
footerName: Подвалда нұсқаулар
footerOrder: 10
---
```

### Негізгі мета қасиеттерін сипаттамасы

- `id`: тіл ішіндегі мақалаға уникалдық идентификатор.
- `title`: браузердегі бет тақырыбы.
- `displayName`: менюде мақаланың атауы.
- `order`: бөлім менюінде мақаланың көрсету тӱрі.
- `published`: мақаланың интерфейсте көрсетілуін белгілейді.
- `historyName` және `historyDescription`: кіру тарихы блогында пайдаланылады.
- `category` және оның қосымша қасиеттері: басты бетте мақаланың көрсетуін белгілейді.
- `headerName` және `headerOrder`: сайт шапкасында мақаланың сілтемесін көрсету.
- `footerName` және `footerOrder`: сайт подвалында мақаланың сілтемесін көрсету.

## Мақаланың Мазмұны

Мета деректерден кейін мақаланың негізгі мазмұны келеді, Markdown-пішімде жазылған. Бұл жерде сіз мәтінді қалыптау үшін Markdown-элементтерді қолдана аласыз,
секілдері: тақырыптар, тізімдер, суреттер, кестелер және т.б.

### Мақала мазмұнының Мисалы

```markdown
# Қолданушының Нұсқаулары

Документацияға қош келдіңіз! Бұл нұсқауда сіз жұмыс істеу мен негізгі функцияларды қалай пайдалануға бастауға болады.

## Жұмысқа бастау

Бастау үшін тіл директориясын жасау және басты бетті сипаттау керек.

<!-- Кейінгі қадамдар түсініктеме... -->
```

## Қолданушыларды қолдану

Мақала мәтінінде `white-label` JSON-жауабынан келетін мәндермен ауыстырылатын айнымалыларды қолдана аласыз.

### Айнымалылар синтаксисті

- Екі құсбелгі фигуралы скобкаларды қолданыңыз: `{{ var }}`.
- Құсбелгілердің ішінде жатқан қасиеттерге қол жеткізіңіз: `{{ company.name }}`.

### Қолдану мисалы

```markdown
Company example компаниясының сайтына қош келдіңіз! Біздің веб-сайтқа қарай: https://company-example-domain.ru.

Біздің партнері: Deep company example (https://deep-company-example-domain.ru)
```

### Өңделгеннен кейін:

```markdown
Company example компаниясының сайтына қош келдіңіз! Біздің веб-сайтқа қарай: https://company-example-domain.ru.

Біздің партнері: Deep company example (https://deep-company-example-domain.ru)
```

## Сілтемелер және навигация

### Мақалаларға сілтеме құрастыру

- **Стандартты сілтемелер**: файл құрылымына негізделген артылым жолын қолданыңыз.
```markdown
[Толығырақ](/ru/documentation/guide)
```


- **`id` пайдалану арқылы қысқа сілтемелер**: мақалалардың идентификаторы бойынша сілтеме жасауға мүмкіндік береді.
```markdown
[Қолданушының Нұсқаулары]([42])
```
