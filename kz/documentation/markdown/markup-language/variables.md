# Markdown-де айдарларды қолдану

Markdown-файлдардан HTML-беттерді жасау процесінде JSON `white-label` деректерінен динамикалық деректерді қойып, айдарларды қолдануға болады.
Бұл сізге мәліметтердің өзгеруіне байланысты автоматты түрде жаңартуға мүмкіндік береді, бастапқы деректерді өңдеу қажеті жоқ, Markdown-файлды өңдеу керек.

## [Сипаттама](description)

**Markdown-де айдарлар не?**

Markdown-де айдарлар `{{ var }}` құрамында көрсетіледі, мұнда `var` — JSON-объектіндеғі мүшелерге жолдың мекенжайы. Бетті жасау кезінде,
бұл айдарлар берілген JSON-объектінен сәйкес келетін мәндермен ауыстырылады.

**Айдарларды қолдануның артықшылықтары:**

- **Мазмұнының динамикалығы**: Бастапқы деректерді өзгергенде беттерде ақпаратты автоматты түрде жаңарту.
- **Қолдауның оңайлығы**: Деректерді өзгертулері тек JSON-объектін жаңартуын қажет етеді, файлдардың көптегенін жаңартуы керек емес.
- **Жиілік**: Қосылған деректерді және көптеген құрылымдарды қолдану мүмкіндігі.

**Айдарлардың синтаксистікі:**

- **Қарапайым айдар**: `{{ variableName }}`
- **Қосылған мүшелер**: `{{ parent.child.property }}`

**Ескерту:**

- JSON-объектінде айдар табылмаса, ол `{{ var }}` ретінде мәтінде қалады, бұл дұрыс көрсетілмейтін көрсетулерге себеп болуы мүмкін.
- Бетті жасау алдында қолданылатын барлық айдарлардың болуын тексеру маңызды.

<br/>

## [Ұсыныстар](recommendations)

- **Деректерді тексеру**: Беттерді жасау алдында JSON-объектінде бар деректерді тексеріңіз.
- **Қателіктерді өңдеу**: Деректерде болмаған деректер немесе қателіктер болған жағдайда жүйенің жұмысын ойлаңыз.
- **Айдарларды документтеу**: Айдарлардың атаулары туралы келісім жасаңыз және олардың қолдануын документтеу арқылы қолдау жасауға мүмкіндік беріңіз.

<br/>

## [Жалғастыру](conclusion)

Markdown-файлдарда айдарларды қолдану компанияға байланысты белгілі компаниядан басқа белгілі компанияға қатынасыз документация мазмұнын қолдану мүмкіндігін береді.


<br/>

## [Қолдану мисалы](example)

**Кадам 1: JSON-деректерді алу**

`white-label` API-сынан алғаныңызды ойлаңыз, келесі JSON-объектін алдыңыз:

```json
{
  "name": "Мысал",
  "domain": "https://example-domain.ru",
  "company": {
    "name": "Компанияның Мысалы",
    "domain": "https://company-example-domain.ru",
    "address": {
      "street": "ул. Мысал, 1",
      "city": "Мәскеу",
      "zip": "123456"
    }
  },
  "deep": {
    "company": {
      "name": "Дерин Компанияның Мысалы",
      "domain": "https://deep-company-example-domain.ru",
      "contacts": {
        "email": "contact@deep-company-example.ru",
        "phone": "+7 (495) 123-45-67"
      }
    }
  }
}
```

**Кадам 2: Перемендерді Markdown-файлында қолдану**

`about.md` Markdown-файлын жасаңыз, осында JSON-объектінен перемендерді қолданамыз:

```md
---
title: {{ company.name }} компаниясы туралы
displayName: {{ company.name }} компаниясы
order: 10
published: true
---

# {{ company.name }} компаниясына қош келдіңіз

Біз сізге **{{ company.name }}** компаниясын таныстырамыз.

## Негізгі ақпарат

- **Компания веб-сайты**: [{{ company.domain }}]({{ company.domain }})
- **Мекен-жай**:
  - Көше: {{ company.address.street }}
  - Қала: {{ company.address.city }}
  - Пошта индексі: {{ company.address.zip }}

## Үйім компаниясы

Біз **{{ deep.company.name }}** ата компаниясын басқарамыз.

- **Веб-сайт**: [{{ deep.company.domain }}]({{ deep.company.domain }})
- **Байланыс**:
  - Email: [{{ deep.company.contacts.email }}](mailto:{{ deep.company.contacts.email }})
  - Телефон: {{ deep.company.contacts.phone }}

## Жұмыс туралы сипаттама

**{{ company.name }}** компаниясы ақпараттық технологиялар саласында жоғары сапалы қызметтер ұсынады.
Мұштерилердің қажеттерін қанағаттандыру үшін біздің өнімдерімізді дамыту және жақсартуға тырысамыз.

## Басқа элементтерде перемендерді қолдану мисалы

- **Компания логотипі**: ![Логотип]({{ company.domain }}/images/logo.png)
- **Бос орынға сілтеме**: [Біздің командамызға қосылыңыз]({{ company.domain }}/careers)
```

**3-шақ: Парақ жасау**

JSON-объектінен айтынмен өзгерістерді қойып Markdown файлын өңдеу кезінде, жүйе барлық `{{ var }}` құрылымдарын қойылған мәндерге ауыстырады.

**Нәтиже:**

Жасалған кейін, `about.md` файлының мазмұны осы түрде болады:

```md
---
title: Компаниялық құрылыс Company Example
displayName: Компаниялық құрылыс
order: 10
published: true
---

# Company Example-ге қош келдіңіз

Біз сізге **Company Example** компаниясын көрсету үшін қуаныштымыз.

## Негізгі ақпарат

- **Компания веб-сайты**: [https://company-example-domain.ru](https://company-example-domain.ru)
- **Мекен-жай**:
  - Көше: ул. Примерная, 1
  - Қала: Мәскеу
  - Пошта индексі: 123456

## Балалы компания

Біз **Deep Company Example** атауымен балалы компанияны басқарамыз.

- **Веб-сайт**: [https://deep-company-example-domain.ru](https://deep-company-example-domain.ru)
- **Байланыс**:
  - Email: [contact@deep-company-example.ru](mailto:contact@deep-company-example.ru)
  - Телефон: +7 (495) 123-45-67

## Жұмыс туралы сипаттама

**Company Example** компаниясы ақпараттық технологиялар саласында жоғары сапалы қызметтер ұсынуға арналған. Біз мүшелеріміздің қажеттерін қанағаттандыру үшін өзіміздің өнімдерімізді дамыту және жақсартуға тырысамыз.

## Басқа элементтерде айтынмендерді пайдалану мисалы

- **Компания логотипі**: ![Логотип](https://company-example-domain.ru/images/logo.png)
- **Бос орынға сілтеме**: [Біздің командамызға қосылыңыз](https://company-example-domain.ru/careers)
```
