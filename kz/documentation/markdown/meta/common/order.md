# order Қасиеті

**Мета қасиеті:** `order`

**Деректілік түрі:** Сан

**Есіктілік аумағы:**
![Қасиеттің әсері](https://raw.githubusercontent.com/SolarSpaceTech/product-documentation-content/refs/heads/main/ru/documentation/markdown/images/order.png)

<br/>

## [Сипаттама](description)

Мета қасиеті `order` бөлімдер мен мақалаларды документация мәзірінде, сондай-ақ бөлімдер ішінде көрсету ретінде белгілеу үшін пайдаланылады.
Бұл қасиет пайдаланушыларға жол табу үшін документацияның құрылымын мана бойынша орналастыруға мүмкіндік береді.

<br/>

### [Негізгі функциялар](basic-functions)

- **Мәзірде ретінде басқару:** Сайт мәзірінде бөлімдер мен мақалаларды көрсету ретінде көрсетілетін кестеленішін белгілейді.
- **Бөлім ішінде ретінде орналастыру:** Ағымдағы бөлім ішінде мақалалардың ретін белгілейді, бұл пайдаланушыларға негізгіден қосымша мазмұндардан бастап көрсету мүмкіндігін береді.
- **Оқу ретінде дәлелдеме:** Пайдаланушыларды документацияны адим адим бойынша өткізу арқылы мана бойынша логикалық ретін белгілейді.

<br/>

### [Басқа мета қасиеттермен қалыптасу](with-other-properties)

- **Бөлімдер үшін:**
  - **`displayName`:** Мәзірде және нан тіркемелерде бөлім атауы.
  - **`published`:** `false` ретінде орнатылса, бөлім және оның мазмұны мәзірде көрсетілмейді, `order` мәніне қаражаттамаса да.

- **Мақалалар үшін:**
  - **`displayName`:** Мәзірде және нан тіркемелерде мақала атауы.
  - **`published`:** `false` ретінде орнатылса, мақала мәзірде және басқа навигациялық элементтерде көрсетілмейді, `order` мәніне қаражаттамаса да.

<br/>

### [Маңызды ескертулер](notes)

- **Міндеттілік:** Мета қасиеті `order` міндетті, элементтердің ретін точны бақылау үшін пайдалану керек.
- **Мәндер ауқымы:** Толық сан мәндерін қабылдау. Қарапайым басқару үшін позитивті сандарды пайдалану үшін.
- **Бірдейлік:** `order` мәндері міндетті түрде бірдей болуы керек емес, бірақ көрсету ретінде белгілеу кезінде айқындықты азайтуға көмектеседі.
- **Сұрыптау:** Элементтер `order` мәнінің өсуі бойынша сұрыпталады. Кішірек мәнді элементтер жоғарыда көрсетіледі.


<br/>

### [Не қалай қолдану керек](when-to-use)

- **Көрсетілу ретін бақылау үшін әрдайым:** Бөлімдер мен мақалаларды белгілі логикалық кестеде көрсету маңызды.
- **Документацияны кеңейту кезінде:** Жаңа материалдарды мүмкін болған орынға қосуға мүмкіндік береді, мазмұндық құрылымдың қорлануын жоюсыз.
- **Пайдаланушы тәжірибесін жақсарту үшін:** Пайдаланушыларға оқиғалы ориентация жасау және қажетті ақпаратты табуға көмектеседі.

<br/>

### [Пайдалану сипаттамалары](advice)

- **Инкременттік қадам:** Жаңа элементтерді қосуға оңай болуы үшін 10 немесе 5 арасында қадамдарды қолданыңыз.
- **Құрылымды басқару:** Бөлімдер мен мақалалардың логикалық кестесін алдын-ала ұйымдастырыңыз.
- **Нумерацияда рет:** Документацияның барлық бөлімдерінде бір нумерация схемасына ұмтылыңыз.
- **Топтау:** Қажет болған жағдайда, жақын мәндерді қолданып, байланысты материалдарды топтауға болады `order` мәндерін пайдалану.
- **Документтау:** Үлкен проекттер үшін `order` мәндерін есептеу, қарым-қатынасыздықтардан сақтану үшін.

<br/>

## [Жалғастыру](conclusion)

`order` мета сипаты, бөлімдер мен мақалаларды документацияда көрсету ретін басқару үшін кіруші инструмент болып табылады.
Ол, логикалық және түсінікті құрылымды жасауға мүмкіндік береді, навигацияны оңайлау және документациямен жұмыс істеу кезінде пайдаланушылардың тәжірибесін жақсартады.

<br/>

order: 2

---
```

order: 3
published: true
---
```

**Мәзірде нәтиже:**

1. Кіріспе
