# ID Қасиеті

**Мета қасиет:** `id`

**Деректер түрі:** Сан

<br/>

## [Сипаттама](description)

`id` қасиеті — бұл документацияда белгілі тілдегі мақаланың уникалды идентификаторы. Мазмұнды ұйымдастыруда кінәлеулердің жасалуын, қысқаған сілтемелерді жасау мен қолданушылардың мақалаларды көру тарихын бақылау мүмкіндігін ұсынады.

<br/>

### [Негізгі функциялар](basic-functions)

- **Уникалды идентификация**: Әр мақаланың өзіне тиесілі уникалды нөмірі бар екенін кепілдейді, бұл кесірліктерді және қиындықтарды алып тастайды.
- **Қысқаған сілтемелер**: `id` пайдалана отырып, мақалаларға қысқаған және есте сілтемелер жасауға мүмкіндік береді.
- **Кіру тарихы**: Қолданушылардың мақалаларды көру тарихын бақылау мен жазу мүмкіндігін ұсынады, бұл қолданушылардың төмендегі тәжірибесін жақсартады.

<br/>

### [Басқа мета қасиеттермен қалыпты іс](with-other-properties)

- **Кіру тарихымен байланыс**: `id` қасиетін `historyName` және `historyDescription` мен қосымша көрсеткіштерімен бірге пайдалану, кіру тарихы блогында жазбаларды құруға мүмкіндік береді.
- **Сілтемелерді жасау**: Мазмұн ішінде сілтемелер жасау кезінде `[Сілтеме мәтіні]([id])` синтаксисті пайдалану, мақаланың саны бойынша `id`-ге ауыстыру. Бұл URL
күзетінің құрылымын өзгерту кезінде навигацияны және сілтемелерді сақтауға көмектеседі.
- **Көптілділік**: `id` тілдік шектер аралығында уникалды болғаны үшін, ол кез келген тілдегі бірдей `id`-лермен бірге біріктірілген мақалаларды ұсынуға мүмкіндік береді, бұл
локализацияны және тілдер арасында ауыстыруды оңайластырады.

<br/>

### [Маңызды ескертулер](notes)

- **Уникалдылық**: `id`-ні бір тілдегі өзгеріссіз болуын кепілдейсіз. `id`-лердің қайталануы сілтемелер мен кіру тарихының дұрыс жұмысын бұзуға себеп болуы мүмкін.
- **Пішімділік**: `id` белгіленгеннен кейін оны өзгертуді міндетті түрде ұсынылмайды, себебі бұл ұсыныстарды және кіру тарихын бұзуы мүмкін.


<br/>

### [Қолдану уақыты](when-to-use)

-	**Навигация**: Статиялар арасында ішкі сілтемелерді `id` негізінде қысқа сілтемелер арқылы жасау үшін.
-	**Локализация**: Көптілділікті бағдарламаларды жасау уақытында, өзінің `id`-сі бар ауылдарының бірдей болуы.

<br/>

### [Пайдалану сұрақтары](advice)

-	**Құжаттама**: Белгіленген `id`-лер мен оларға қосымша статияларды есептеңіз.
-	**Сілтемелерді жаңарту**: Статияның `id`-сін өзгерту кезінде (шiddetle ұсынылмайды) оның сілтемелерін жаңарту керек.
-	**Статияны жою**: Статияны жою кезінде босатылған `id`-ді пайдалануға ұсынылмайды.

<br/>

## [Жалғастыру](conclusion)

`id` қасиеті документацияда мазмұнды басқару және басқару үшін фундаменталды мета-қасиет болып табылады. `id`-ді дұрыс пайдалану навигацияны оңтайландырады,
пайдаланушы тәжірибесін жақсартады және жүйенің қолданылуын қамтамасыз етуге көмектеседі.

<br/>

## [Қолдану мисалдары](examples)

### Мисал 1: Статияға қысқа сілтеме жасау

Статия файлы: `ru/documentation/guide.md`

```markdown
---
id: 42
title: Пайдаланушының нұсқасы
displayName: Нұсқа
published: true
historyName: Нұсқа
historyDescription: Пайдаланушының нұсқасы сипаттамасы
---
# Пайдаланушының нұсқасы

Пайдаланушының нұсқасына қош келдіңіз.
```

Статия файлы: `ru/documentation/new-guide.md`

```markdown
---
id: 43
title: Жаңа пайдаланушының нұсқасы
displayName: Жаңа нұсқа
published: true
historyName: Жаңа пайдаланушының нұсқасы
historyDescription: Жаңа пайдаланушының нұсқасы сипаттамасы
---
# Жаңа пайдаланушының нұсқасы

Жаңа пайдаланушының нұсқасына қош келдіңіз.

Басқа статияда қысқа сілтемені пайдалану:

Ескі нұсқаны алу үшін [Нұсқа]([42]) көріңіз.
```

Сипаттама:

-	`[42]` рендерленгенде `/ru/documentation/guide` сілтемесіне айналды, сондықтан `[Нұсқа](/ru/documentation/guide)` алынады.
-	Егер `guide.md` файлдың мекенжайы немесе атауы өзгерсе, сілтеме дұрыс қалады, себебі ол `id`-ге негізделген.

<br/>

### Мисал 2: Қолдану тарихын көру

-	Статьяның `id: 42` бетіне кіретін кезде, оның туралы ақпаратты кіру тарихы блогына қосылады, көрсету `historyName` және `historyDescription` негізінде құрылады.
-	Бұл пайдаланушыға соңғы көруді қараган мақалаларға оңай қайта оралу мүмкіндігін береді.
