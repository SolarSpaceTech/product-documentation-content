# headerOrder Қасиеті

**Мета қасиет:** `headerOrder`

**Деректілік түрі:** Сан

**Есіктік аудармасы:**
![Қасиеттің әсері](https://raw.githubusercontent.com/SolarSpaceTech/product-documentation-content/refs/heads/main/ru/documentation/markdown/images/header-order.png)

<br/>

## [Сипаттама](description)

`headerOrder` мета қасиеті, документация шапкасындағы сілтемелерді көрсету ретін белгілеу үшін пайдаланылады. Шапка (header) навигацияның негізгі элементі ретінде қызмет етеді, пайдаланушыларға ең маңызды бөлімдерге немесе статияларға жылдам доступ береді.
`headerOrder` арқылы, шапкадағы сілтемелердің тізімін дұрыс баптауға болады, пайдаланушыларға мансапты және ықтылы навигация қамтамасыз етеді.

<br/>

### [Негізгі функциялар](basic-functions)

- **Шапкадағы сілтемелердің тізімін басқару:** `headerOrder` мәні статия сілтемесінің шапкасындағы орнын белгілейді.
`headerOrder` мәні кішірекке (`headerOrder` мәнімен байланысты, экран резолюциясына байланысты) немесе жоғарыға көрсетіледі.
- **Көрсету маңыздылығы:** Пайдаланушыларға ең маңызды мазмұны бірінші орынға қойып, сілтемелерді маңыздылығы бойынша орналастыру мүмкіндігін береді.
- **Теңдікті баптау:** Шапка құрылымын басқару мүмкіндігін ұсынады, оны документацияның қажеттеріне қолданысқа сәйкес адаптациялауға мүмкіндік береді.

<br/>

### [Басқа мета қасиеттермен қалай жұмыс істейді](with-other-properties)

- **`headerName` бірге:** Сілтемені шапкада көрсету үшін `headerName` атауын көрсету қажет, ал `headerOrder` оның орнын белгілейді.
- **`published` бірге:** `published` мәні `false` болса, статия `headerName` және `headerOrder` мәндеріне қарап тапсырмаса да, шапкада көрсетілмейді.
- **Басқа мета қасиеттермен:** Мәзірде статияны көрсету үшін, менюде, төменгі жолда немесе басқа интерфейс элементтерінде (`order` немесе `footerOrder` сияқты) `headerOrder` орналасуына әсер етпейді.


<br/>

### [Маңызды ескертулер](notes)

- **Міндеттілік:** Мақаланы көрсету үшін шапкада `headerName` және `headerOrder` қасиеттері көрсетілген болуы керек. Олардың бірі жоқ болса, сілтеме шапкада көрсетілмейді.
- **Құқық ауқымдылығы:** `headerOrder` бүтін сандарды қабылдайды. Түсініктілік және басқарудың оңайлығы үшін мүмкіндіктің артықшылығын пайдалану мүлдем ұсынылады.
- **Мәндердің бірдейлігі:** `headerOrder` мәндерінің бірдей болуы міндетті емес, бірақ көрсетілгендердің тізімін бақылау үшін `headerOrder` мәндерінің бірдей болуы ұсынылады.
- **Сұрыптау:** Шапкадағы сілтемелер `headerOrder` мәнінің артуы бойынша сұрыпталады. Кіші мәнді сілтемелер алдында көрсетіледі.

<br/>

### [Не уақытта пайдалану керек](when-to-use)

- **Документация шапкасын қайта орнату кезінде:** Сілтемелерді көрсету тізімін белгілеу және мүмкіндіктердің логикалық навигациясын қамтамасыз ету үшін.
- **Мазмұндықты басқару үшін:** Сілтемелерді маңыздылық бойынша орналастыру қажет болған кезде пайдалану.
- **Документацияны жаңарту кезінде:** Шапкада жаңа сілтемелерді қосу үшін мазмұндықтың барлық тізімін бұзбауға болмаса да, барлық тізімдің төмендегі тәртібін бұзбауға болады.

<br/>

### [Пайдалану туралы мақсаттар](advice)

- **Жаңа сілтемелерді қосу үшін арттыру қадамы:** `headerOrder` мәндері арасында 5 немесе 10 арттыру қадамын пайдаланыңыз.
- **Маңыздылықты есептеу:** Ең маңызды сілтемелерді шапканың басына орналастырыңыз, оларға кіші мәндер беріңіз.
- **Тексеру:** Сілтемелерді көрсету кезінде көмекші құрылғылар мен экрандарда қалай көрсетілетінін тексеріңіз, орналасу тәртібі мен пайдалану оңайлығын тексеріңіз.
- **Құрылымды жоспарлау:** Шапкадан қолжетімді болуы керек бөлімдер немесе мақалаларды алу үшін алдымен ойлаңыз, олардың төмендегі тәртібін белгілеңіз.
- **Бірдейлік:** Нумерация схемасы мен атаулар стиліне сенімді болыңыз.

## [Жалғастыру](conclusion)

`headerOrder` мета қасиеті сілтемелерді шапкада көрсету төмендегі тізімін басқару үшін кінәлі құралды қамтамасыз етеді.
Осы қасиетті дұрыс пайдалану арқылы логикалы және интуитивті навигация жасау, пайдаланушы тәжірибесін жақсарту және маңызды ақпаратқа қатысты қолжетімділікті арттыруға мүмкіндік береді. Шапканың құрылымын жоспарлаңыз және интерфейсін теңшеу үшін `headerOrder` және `headerName` көмегімен пайдаланыңыз.


<br/>

## [Қолдану мисалдары](examples)

### Мисал 1: Тізімнің шапкасындағы сілтемелерді тарату

**Мақала 1:**

```md
---
title: Жұмысқа шығу
displayName: Таным
order: 1
published: true
headerName: Жұмысқа шығу
headerOrder: 10
---
# Жұмысқа шығу

Біздің жұмысқа шығу нұсқауларына қош келдіңіз...
```

**Мақала 2:**

```md
---
title: API Документация
displayName: API
order: 2
published: true
headerName: API
headerOrder: 20
---
# API Документация

Осы бөлімде API үшін қол жетімді әдістер сипатталған...
```

**Мақала 3:**

```md
---
title: Қолдау
displayName: Қолдау
order: 3
published: true
headerName: Қолдау
headerOrder: 30
---
# Қолдау

Сізде сұрақтар туындаса, біздің қолдау қызметіне хабарласыңыз...
```

**Документация шапкасындағы нәтижесі:**

1. Жұмысқа шығу (headerOrder: 10)
2. API (headerOrder: 20)
3. Қолдау (headerOrder: 30)

<br/>

### Мисал 2: `headerOrder` мәнін бос қалдыру

**Мақала:**

```md
---
title: Блог
displayName: Блог
order: 4
published: true
headerName: Блог
---
# Блог

Соңғы жаңалықтар мен жаңартуларды оқыңыз...
```

**Нәтиже:** `headerOrder` мәні көрсетілмеген себебінен, "Блог" мақаласының сілтемесі документация шапкасында көрсетілмейді.
