# footerOrder Қасиеті

**Мета қасиеті:** `footerOrder`

**Деректілік түрі:** Сан

**Есепке алу аумағы:**
![Қасиеттің әсері](https://raw.githubusercontent.com/SolarSpaceTech/product-documentation-content/refs/heads/main/ru/documentation/markdown/images/footer-order.png)

<br/>

## [Сипаттама](description)

Мета қасиеті `footerOrder`, документацияның астығындағы мақалаларға сілтемелерді көрсету ретін белгілеу үшін пайдаланылады. Астық (footer) навигацияның негізгі элементі ретінде қызмет етеді,
пайдаланушыларға ең маңызды бөлімдерге немесе мақалаларға жылдам қатынау мүмкіндігін береді. `footerOrder` арқылы сіз астықтағы сілтемелердің тізімін дәлдік және пайдалы навигацияны ұсынуға мүмкіндік бересіз.

<br/>

### [Негізгі функциялар](basic-functions)

- **Сілтемелердің тізімін басқару:** `footerOrder` мәні документацияның астығындағы мақаланың сілтемесінің орнын белгілейді. 
`footerOrder` мәні кішірек болған сілтемелерді көрсетеді (экран ретінен байланысты).
- **Көрсету үрдісі:** Пайдаланушыларға ең маңызды мазмұны бірінші орынға ұсыну арқылы сілтемелерді орналастыру мүмкіндігін береді.
- **Теңдікті баптаушылық:** Астықтың құрылымын басқару мүмкіндігін ұсынады, ол документацияның қажеттеріне қауіпсіздік береді.

<br/>

### [Басқа мета қасиеттерімен қалай жұмыс істейді](with-other-properties)

- **`footerName` бірге:** Астықта сілтемені көрсету үшін `footerName` атауын көрсету қажет, ал `footerOrder` оның орнын белгілейді.
- **`published` бірге:** `published` мәні `false` болса, мақала астықта көрсетілмейді, `footerName` және `footerOrder` мәндеріне қауіпсіздік берілмейді.
- **Басқа мета қасиеттерімен:** Мәтіндікті көрсету, астықта немесе басқа интерфейс элементтерінде (`order` немесе `footerOrder`) көрсету үшін мақала көрсету үшін әсер етпейді.


<br/>

### [Маңызды ескертулер](notes)

- **Міндеттілік:** Мақаланы көрсету үшін астыға сілтемені көрсетуі керек: `footerName` және `footerOrder`. Олардың бірі жоқ болса, сілтеме астыға көрінбейді.
- **Мәндер ауқымы:** `footerOrder` бүтін сандарды қабылдайды. Түсініктілік және басқарудың оңайлығы үшін мүмкіндіктің артықшылығынан позитивті сандарды қолдану мүлкін.
- **Мәндердің бірденділігі:** `footerOrder` мәндерінің бірденділігі міндетті емес, бірақ сілтемелердің көрсету ретін белгілеу үшін `footerOrder` мәндерінің бірденділігін ұсынылатын керек.
- **Сұрыптау:** Астыға сілтемелер `footerOrder` мәнінің артуы бойынша сұрыпталады. Кіші мәнді сілтемелер бірінші рет көрсетеді.

<br/>

### [Не уақытта пайдалану керек](when-to-use)

- **Документацияның астығын қайта орнату кезінде:** Сілтемелерді көрсету ретін белгілеу және логикалық навигацияны қамтамасыз ету үшін.
- **Мазмұндықты басқару үшін:** Сілтемелерді маңыздылық бойынша орналастыру қажет болған кезде пайдалану.
- **Документацияны жаңарту кезінде:** Астыға жаңа сілтемелерді қосу үшін мазмұндықтың барлық ретін бұзбауға болмай, сілтемелердің белгілі ретін сақтау үшін.

<br/>

### [Пайдалану туралы кеңес](advice)

- **Жаңарту адымы:** `footerOrder` мәндері арасында 5 немесе 10 аралығында адымдар қосу үшін пайдаланыңыз.
- **Маңыздылықты есептеу:** Ең маңызды сілтемелерді астығында орналастырып, оларға кішірек `footerOrder` мәндерін беру.
- **Тексеру:** Сілтемелерді көрсетуін көздердің және экрандардың арасында тексеріңіз, көрсету реті мен пайдалану қоғамының дұрыстығын тексеріңіз.
- **Мәндер арасында аралық:** Сілтемелердің `footerOrder` мәндері арасында жаңа сілтемелерді қосу мүмкіндігі үшін (мысалы, 10, 20, 30) аралықты пайдаланыңыз, барлық элементтердің нумерациясын қайта жасау қажеті жоқ.
- **Құрылымды жоспарлау:** Астыға не немесе мақалалардың қайсыларына қол жеткізуі керек пайдалануға, олардың ретін белгілеуіңіз керек.
- **Бірдейлік:** Нумерация схемасы мен атаулар стиліне сенімді болыңыз.

<br/>

## [Жалғастыру](conclusion)

`footerOrder` мета мүмкіндігі сілтемелерді астығын басқару үшін кіру жолын белгілеу маңызды құрал. Бұл мүмкіндікті дұрыс пайдалану арқылы логикалық және интуитивті навигацияны құруға мүмкіндік береді, пайдаланушы тәжірибесін жақсартады және маңызды ақпаратқа қолжетімділік пен қолайлылықты ұсынады. Астығының құрылымын жоспарлау және `footerName` бірлікті `footerOrder` пайдалану интерфейсін тиімді баптау үшін пайдаланыңыз.


<br/>

## [Қолдану мисалдары](examples)

### Мисал 1: Түсініктеме сілтемелердің тізімін орнату

**Мақала 1:**

```md
---
title: Жұмысқа шығу басталуы
displayName: Таным
order: 1
published: true
footerName: Жұмысқа шығу
footerOrder: 10
---
# Жұмысқа шығу басталуы

Біздің жұмысқа шығу нұсқауларына қош келдіңіз...
```

**Мақала 2:**

```md
---
title: API Документация
displayName: API
order: 2
published: true
footerName: API
footerOrder: 20
---
# API Документация

Осы бөлімде қол жетімді API әдістері сипатталған...
```

**Мақала 3:**

```md
---
title: Қолдау
displayName: Қолдау
order: 3
published: true
footerName: Қолдау
footerOrder: 30
---
# Қолдау

Сізде сұрақтар туындаса, қолдау қызметімізге хабарласыңыз...
```

**Документацияның астындағы нәтиже:**

1. Жұмысқа шығу (footerOrder: 10)
2. API (footerOrder: 20)
3. Қолдау (footerOrder: 30)

<br/>

### Мисал 2: `footerOrder` мәнін бос қалдыру

**Мақала:**

```md
---
title: Блог
displayName: Блог
order: 4
published: true
footerName: Блог
---
# Блог

Соңғы жаңалықтар мен жаңартуларды оқыңыз...
```

**Нәтиже:** `footerOrder` мәні көрсетілмеген себепті, "Блог" мақаласының сілтемесі документацияның астында көрсетілмейді.
